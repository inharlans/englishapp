# 모바일-웹 패리티 2차 계획 (안정화/확장)

Back to [AGENTS.md](../AGENTS.md)

## 목표

- 1차 핵심 기능을 유지하면서 회귀 방지 체계를 고정한다.
- 기능 확장보다 계약 안정성과 운영 검증 자동화에 우선순위를 둔다.
- 결제 기능은 웹 준비 완료 전까지 제외한다.

## 범위

- API 계약 drift 감지 및 회귀 테스트 자동화
- 모바일-웹 교차 검증 시나리오 정착
- 학습/마켓/클리퍼 확장 기능의 안정화
- 장애 대응 기준(권한/데이터 불일치 최우선) 운영화

## 실행 체크리스트

- [ ] 모바일 API wrapper 계약 테스트(응답 shape 변형 케이스 포함)
- [ ] 404 Known Gap 재발 감시 스크립트 추가
- [ ] 웹 작업 -> 모바일 반영, 모바일 작업 -> 웹 반영 교차 테스트 자동화
- [ ] 릴리즈 전 고정 점검 루틴 문서화
- [ ] 1차 기능 회귀 0건 유지

## 통과 기준

- 1차 핵심 플로우 PASS 유지
- 2차 확장 항목 PASS율 100%
- 반복 실행(최소 3회)에서 결과 일관성 확보
