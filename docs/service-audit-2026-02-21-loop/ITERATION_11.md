## 점검 범위

- 결제 흐름 및 결제 API는 제외

## 발견/반영 체크리스트 (10건)
1. 다운로드 버전 로그 집계가 반복 필터(`logs.filter`) 기반이라 카드 수가 늘면 비용이 증가함 -> 단어장별 로그 그룹 맵으로 1회 분류 후 집계하도록 최적화
2. 무료 사용자에게 잔여 다운로드 가능 단어 수가 바로 보이지 않음 -> `남은 다운로드 용량` 수치 노출
3. 업데이트 가능한 다운로드 단어장이 있어도 상단에서 즉시 인지하기 어려움 -> 상태 배너(`role=status`, `aria-live`) 추가
4. 페이지 루트에 제목 연결 랜드마크가 없어 스크린리더 내비게이션이 불명확함 -> `aria-labelledby` + 제목 id 연결
5. 내가 만든 단어장 섹션 제목-영역 연결이 약함 -> 섹션 `aria-labelledby` 보강
6. 다운로드 단어장 섹션 제목-영역 연결이 약함 -> 섹션 `aria-labelledby` 보강
7. 카드 제목 링크/카드 클릭 링크의 목적이 제목 텍스트에만 의존함 -> `aria-label`로 이동 목적 명시
8. 내가 만든 단어장 카드에서 최근 수정 시점이 보이지 않아 최신성 판단이 어려움 -> `최근 수정` 날짜 표시 추가
9. 다운로드 단어장 카드에서도 최신성 기준(원본 수정일)이 약함 -> `최근 수정일` 표시 추가
10. 다운로드 단어장 빈 상태에서 오프라인 페이지 진입 동선이 없음 -> 보조 CTA(`오프라인 보기`) 추가
