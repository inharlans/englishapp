## 실플레이 5사이클 기록
1. 사이클 1: 오답 제출(`ofx`) 후 `다음` 이동 흐름 점검
   - 이슈: 오답 큐 재출제/파트 전환 경계에서 상태 오염 가능성 확인
2. 사이클 2: 오답 재출제 후 재오답(`zzz`)으로 큐 누적 상황 점검
   - 이슈: 파트 전환 시 오답 큐가 새 파트 출제에 섞일 여지 확인
3. 사이클 3: 파트 전환(1파트 -> 2파트) 후 즉시 출제 데이터 확인
   - 이슈: 큐/파트 상태 분리 필요
4. 사이클 4: 진행률/남은 문제 수치 점검
   - 이슈: `현재 파트 남은 문제`가 시도 횟수 기반으로 감소해 체감과 불일치
5. 사이클 5: 단축키/피드백 가독성 점검
   - 이슈: `Enter` 다음 이동, `R` 오답 재시도, `Esc` 입력 비우기 등 빠른 조작 보강 필요

## 적용한 개선점 (10+)
1. 파트 전환 공통 리셋 함수(`resetPartScopedState`) 도입
2. 파트 전환 시 오답 큐/오답 큐 ref 즉시 초기화
3. 파트 전환 시 피드백/메시지/파트 시도 카운트 일괄 초기화
4. 파트 크기 변경 시 파트 1로 이동하면서 파트 스코프 상태 초기화
5. `[` `]` `Home` `End` 키 이동도 공통 리셋 함수 사용
6. 피드백 상태에서 `Enter`로 다음 문제 이동 추가
7. 오답 피드백 상태에서 `R` 키로 즉시 재시도 추가
8. 풀이 중 `Esc`로 현재 입력값 비우기 추가
9. 단어 퀴즈에서 문제 의미/예문/예문해석 출력에 텍스트 정제 적용
10. 파트 진행률 계산을 시도 횟수 대신 `고유 풀이 수` 기준으로 변경
11. 상태 바에 `고유 풀이`, `시도 횟수`를 분리 표기해 혼동 완화
12. WORD 모드 오답 피드백은 정답 단어 중심으로 표시(의미는 MEANING 모드에만 병기)
13. 단축키 안내 문구를 실제 동작(`Enter`, `R`, `Esc`)에 맞게 보강
14. WORD 모드 입력 힌트에 정규화 안내(대소문자/공백) 추가
