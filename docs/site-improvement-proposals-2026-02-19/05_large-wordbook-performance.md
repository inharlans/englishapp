# 제안서 05: 대용량 단어장 성능/모바일 가독성 개선

## 배경
- 600개 이상 단어장에서 상세 페이지가 매우 긴 목록을 한 번에 렌더링.
- 모바일에서 스크롤/탐색 부담이 크고 초기 렌더 지연 체감이 큼.

## 문제
- 초기 렌더 비용 증가로 상호작용 지연 가능.
- 원하는 단어를 찾기까지 스크롤 거리가 길어 학습 집중도가 떨어짐.

## 제안
- 단어 목록에 가상 스크롤(virtualized list) 적용.
- 기본 페이지네이션 또는 “상위 N개 우선 표시 + 더 보기”로 분할.
- 빠른 필터 추가: 품사/오답우선/최근학습.
- 모바일 전용 밀도 프리셋(카드 간격 축소 + 탭/입력 고정).

## 구현 범위
- 대상: `app/wordbooks/[id]/page.tsx` 단어 목록 영역
- API: 필요한 최소 필드만 지연 로딩하는 옵션 추가

## 성공 지표
- 상세 페이지 첫 상호작용 시간(TTI) 개선
- 모바일 세션당 평균 스크롤 거리 감소
