# 제안서 01: 의미/품사 렌더링 품질 정상화

## 배경
- 단어장 상세/암기 화면에서 의미가 `동사같다`, `전치사...에`처럼 붙어서 표시됨.
- 사용자 입장에서는 데이터 품질 문제가 UI에서 더 심하게 보임.

## 문제
- 품사 태그가 텍스트로 노출될 때 시각적 구분이 없어 가독성이 낮음.
- 의미 구분자(`,`, 태그 전환) 기준 UI 분리가 불안정해 학습 피로가 큼.

## 제안
- 공통 파서(`MeaningView` 확장)로 `(명)/(동)...` 태그를 칩 형태로 렌더링.
- 의미 항목은 `태그 + 뜻` 구조로 정규화해 리스트 렌더링.
- `...`/불필요 기호를 화면에서 숨기거나 보조표기로 다운그레이드.

## 구현 범위
- 대상: `app/wordbooks/[id]`, `app/wordbooks/[id]/memorize`, `app/wordbooks/[id]/quiz-*`
- 공통 유틸 추가: 의미 문자열 파싱/토큰화 모듈
- 회귀 테스트: 태그/구분자별 스냅샷 테스트

## 성공 지표
- 의미 렌더링 관련 사용자 제보 건수 50% 이상 감소
- 암기 화면 체류 시간/완주율 개선
