# 제안서 04: 신고/차단 UX 명확화

## 배경
- 상세/마켓에서 `블랙` 버튼 표기가 노출됨.
- 용어가 모호하고 행동 결과(차단 범위/해제 방법)가 사용자에게 명확하지 않음.

## 문제
- 기능은 맞아도 라벨/설명이 불명확하면 오작동 체감 발생.
- 신고와 차단의 목적이 분리되지 않아 관리 액션 의도가 흐려짐.

## 제안
- 버튼 라벨을 `차단`으로 통일.
- 클릭 시 설명 툴팁/모달:
  - “이 제작자의 단어장을 마켓에서 숨깁니다”
  - “내 설정에서 언제든 해제 가능”
- 신고/차단 액션 이후 토스트/undo 제공.

## 구현 범위
- 대상: `ReportWordbookButton`, `BlockOwnerButton`, 마켓/상세 페이지
- 용어 통일 가이드(UX 카피 룰) 추가

## 성공 지표
- 차단/신고 관련 오해성 문의 감소
- 차단 해제 경로 도달률 상승
