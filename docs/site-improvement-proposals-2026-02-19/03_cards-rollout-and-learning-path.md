# 제안서 03: 카드 학습 모드 롤아웃 + 학습 동선 통합

## 배경
- 카드 학습 페이지(`/wordbooks/[id]/cards`)는 로컬 구현 대비 배포본에서 404 확인됨.
- 사용자 관점에서 “암기/퀴즈/카드”의 역할 차이가 명확하지 않음.

## 문제
- 기능 존재 여부가 환경마다 다르면 사용자 혼란/문의 증가.
- 학습 루프(암기→퀴즈→복습)에서 카드 모드가 자연스럽게 노출되지 않음.

## 제안
- 카드 모드를 정식 탭으로 고정 노출 + `마지막 학습 이어서` 지원.
- 배포 전 가드:
  - 라우트 헬스체크(E2E)에서 `/cards` 200 확인
  - 404 감지 시 탭 자동 숨김(feature flag fallback)
- 세션 리캡 패널에서 다음 추천 동선에 카드 모드 포함.

## 구현 범위
- 대상: 학습 탭, Resume 버튼, CI E2E, feature flag
- 배포 체크리스트에 “신규 학습 라우트 확인” 항목 추가

## 성공 지표
- 카드 탭 유입률/재방문률 증가
- 404 관련 사용자 제보 0건 유지
