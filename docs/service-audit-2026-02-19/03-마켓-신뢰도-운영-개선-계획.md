# 마켓 신뢰도/운영 개선 계획

## 배경

- 마켓/내 다운로드 목록에 테스트성 데이터(E2E 성격 제목, 내부 seed 설명)가 사용자에게 그대로 노출된다.
- 콘텐츠 출처 정보가 개발용 텍스트 형태로 표시되어 서비스 완성도가 낮게 보일 수 있다.

## 목표

1. 사용자 노출 데이터와 내부 테스트 데이터를 완전히 분리한다.
2. 마켓 상세/목록의 메타데이터를 사용자 친화 문구로 정제한다.
3. 신고/블랙리스트/관리자 검수 흐름을 운영 표준으로 고정한다.

## 구현 계획

### 1) 노출 정책 정비

- `isPublished=true`만으로 노출하지 말고 `isDiscoverable=true` 같은 추가 조건 도입
- 테스트/시드/내부 검증용 단어장은 기본 `isDiscoverable=false`
- 관리자만 볼 수 있는 내부 목록과 일반 마켓 목록 분리

### 2) 메타데이터 정제

- `Imported from ... generated.tsv` 형태 문구는 관리자 전용 정보로 이동
- 사용자 화면에는 출처를 간단한 레이블(예: "공식/커뮤니티")로 표기
- 이메일 주소 전면 노출 대신 닉네임/마스킹 정책 적용 검토

### 3) 신고-검수-조치 루프

- 신고 유형별 기본 SLA 정의
- 관리자 페이지에 신고 상태(접수/검토중/해결/기각) 추가
- 누적 신고 임계치 도달 시 임시 비노출 자동화

## 테스트 계획

1. 테스트 데이터가 마켓에서 비노출되는지 확인
2. 일반 사용자/관리자 계정에서 목록 차등 노출 확인
3. 신고 접수 후 관리자에서 상태 변경, 결과 반영까지 E2E 점검

## 완료 기준(Definition of Done)

- 내부 테스트성 단어장 0건 노출
- 마켓 카드/상세 텍스트가 사용자용 문구로 통일
- 신고 처리 상태를 관리자에서 추적 가능

## 롤아웃

- 1단계: 신규 업로드부터 정책 적용
- 2단계: 기존 노출 데이터 일괄 재분류
- 3단계: 신고 자동화 룰 적용
