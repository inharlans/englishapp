# 로그인 이후 실행 점검 리포트 (MCP)

작성일: 2026-02-19  
계정 상태: 로그인 성공(PRO, 관리자 권한 계정)  
점검 범위: 로그인 이후 전용 기능

## 점검한 기능과 결과

1. 단어장 생성(수동 입력)
- 결과: 성공
- 확인 내용: `+ row` 입력, 유효성(valid/invalid), 생성 후 상세 이동 정상

2. 단어장 공개/비공개 전환
- 결과: 성공
- 확인 내용: `공개로 전환` 클릭 시 상태가 즉시 `공개`로 변경됨

3. 단어 학습 탭(암기/의미퀴즈/단어퀴즈)
- 결과: 기본 동작 성공
- 확인 내용: 정답/오답 판정 및 목록 반영 확인

4. 정답/오답/회복 목록
- 결과: 성공
- 확인 내용: 퀴즈 결과가 목록 상태(`CORRECT`, `WRONG`)에 반영됨

5. 오프라인 저장/오프라인 학습
- 결과: 동작은 성공, 콘텐츠 품질 이슈 재현
- 확인 내용: 오프라인 저장 후 `/offline/wordbooks/:id` 진입 가능
- 이슈: 뜻 보기에서 `??` 노출 재현

6. 신고 UI
- 결과: 폼 오픈/닫기 성공
- 확인 내용: 사유 선택/상세 입력/제출 버튼 표시 확인

7. 블랙리스트 페이지
- 결과: 성공
- 확인 내용: 목록/빈 상태 안내 정상

8. 관리자 페이지
- 결과: 접근 성공
- 확인 내용: 사용자 요금제/관리자 토글 UI, 신고 섹션, 관측성 섹션 렌더링
- 관찰 이슈: 상단 일부 버튼이 비활성 상태(`새로고침 중...`)로 고정된 듯한 표현

9. 요금제 페이지(로그인 상태)
- 결과: 렌더링 성공
- 확인 내용: 현재 요금제/정책/가격 노출 정상
- 관찰 이슈: `Logged-in email` 등 영어 문구 혼재

## 발견 이슈 요약

- P0: 오프라인 포함 텍스트 깨짐(`??`) 노출
- P1: 한국어 UI에서 영어 문구 혼재(`Logged-in email`, 기존 `Invalid credentials.` 케이스)
- P1: 마켓/목록에 내부용 텍스트 노출 가능성(테스트/seed성 데이터)
- P2: 관리자 화면 일부 버튼 상태 표현 개선 필요(로딩 고정 의심)

## 즉시 권장 조치

1. 텍스트 품질 이슈를 최우선으로 정리(인코딩 + 데이터 정제 + UI fallback)
2. 사용자 노출 문구 한국어 일괄 점검(인증/요금제/오류 메시지 포함)
3. 마켓 노출 정책 강화(테스트 데이터 차단)
4. 관리자 관측성/버튼 상태 로직 점검
