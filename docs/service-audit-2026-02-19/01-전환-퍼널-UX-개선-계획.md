# 전환 퍼널 UX 개선 계획

## 배경

- 비로그인 상태에서 `요금제` 클릭 시 즉시 가격 정보 대신 로그인으로 유도되는 구간이 존재한다.
- 결제 전 사용자의 의사결정(가격/혜택 비교) 전에 인증을 강제하면 이탈이 증가할 수 있다.

## 목표

1. 비로그인 사용자도 `요금제` 페이지 핵심 정보는 즉시 확인 가능하게 한다.
2. 실제 결제 직전 단계에서만 로그인/인증을 요구하도록 경계를 명확히 한다.
3. 로그인 이후 원래 의도한 결제 플로우로 정확히 복귀시킨다.

## 구현 계획

### 1) 접근 정책 분리

- 미들웨어/라우팅에서 `/pricing`은 공개 접근 허용.
- 결제 실행 API/버튼 클릭 시점에서 인증 체크.

예상 수정 지점:
- `middleware.ts`
- 결제 시작 API 라우트(`app/api/.../route.ts` 계열)
- 요금제 페이지 컴포넌트(`app/pricing/...`)

### 2) CTA 구조 변경

- 비로그인: `로그인 후 결제하기` 버튼 노출
- 로그인: `월간 결제`, `연간 결제` 버튼 즉시 노출
- 버튼 클릭 시 `next` 파라미터로 복귀 경로 강제

### 3) 메시지 정비

- 가격/환불/자동갱신/해지 시점 안내를 한국어로 명확히 표기
- 로그인 필요 시 이유를 짧게 설명(결제 계정 식별 목적)

## 테스트 계획

1. 비로그인으로 `/pricing` 진입 가능 확인
2. 비로그인 상태 결제 버튼 클릭 -> 로그인 이동 + `next=/pricing` 유지
3. 로그인 완료 후 `/pricing` 복귀 및 결제 버튼 정상 노출 확인
4. 기존 결제 성공/취소 리다이렉트(`?payment=success|cancel`) 회귀 확인

## 완료 기준(Definition of Done)

- 비로그인 사용자가 가격표/혜택/FAQ를 로그인 없이 열람 가능
- 결제 기능은 인증 사용자만 실행 가능
- E2E 시나리오에 게스트->로그인->결제 진입 경로가 추가되고 CI 통과

## 롤아웃

- 1차: 기능 플래그로 공개(내부 1일 모니터링)
- 2차: 전면 적용
- 지표: `/pricing` 진입 대비 결제 시도율, 로그인 전 이탈률
