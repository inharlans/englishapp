# 실서비스 진단 요약 (MCP 점검)

작성일: 2026-02-19  
대상: `https://www.oingapp.com`  
점검 방식: MCP 브라우저 수동 사용(비로그인 사용자 흐름 중심)

## 핵심 결론

1. 결제/전환 이전 단계 UX는 동작하지만, 공개 페이지 접근 흐름(요금제)에서 마찰이 있다.
2. 단어장 상세에서 `??`, `??? ???` 같은 깨진/의미 손실 텍스트가 노출되어 신뢰도 저하가 크다.
3. 마켓에 내부 테스트성 데이터(E2E 제목군)가 노출되어 콘텐츠 품질 인식이 떨어진다.
4. 로그인 오류 문구가 영어(`Invalid credentials.`)로 노출되어 한국어 일관성 정책과 충돌한다.
5. 운영 측면에서 크론 실패 알림 이력이 있었으므로, 실패 원인 가시성과 재시도 정책을 강화해야 한다.

## 우선순위

- P0: 텍스트 품질/인코딩 문제(`??` 노출) 제거
- P1: 마켓 노출 정책 정비(테스트 데이터 비노출)
- P1: 로그인/오류 문구 한국어 통일
- P2: 비로그인 요금제 접근 UX 개선
- P2: 크론 워크플로 관측성/알림 체계 정비
