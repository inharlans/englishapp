# 웹사이트 개선안 (현장 순회 기반) - 2026-02-23

## 1) 점검 방식
- 브라우저 순회: Playwright MCP
- 점검 경로: `/`, `/wordbooks/market`, `/pricing`, `/offline`, `/login?next=/wordbooks`
- 뷰포트: 데스크톱 + 모바일(390x844)
- 적용 관점(스킬 기준):
  - `nextjs-frontend-guidelines`: 정보구조/접근성/Next 라우팅 흐름
  - `frontend-design`: 메인 랜딩의 시각 구조/브랜딩 일관성
  - `web-design-guidelines`: UI 가독성/전환 동선/마이크로카피
  - `vercel-react-best-practices`: 렌더링 성능/리스트 상호작용

## 2) 핵심 진단 요약

### P0 (즉시 수정)
1. 로그인 패널 카피 오류
- 현상: `다시 당신 것을 환영합니다` 문구가 메인/로그인에 노출됨.
- 문제: 비문으로 신뢰감 저하.
- 수정 대상: `components/auth/LoginPanel.tsx`
- 제안 카피: `다시 오신 것을 환영합니다` 또는 `학습을 계속해 보세요`.

2. 홈 화면 정보 과밀 + 로그인 폼 동시 노출
- 현상: 메인에서 설명 카드 + 지표 + 로그인 폼 + 사업자 정보가 한 화면에 누적.
- 문제: 첫 액션(학습 시작/로그인)이 분산되어 전환 집중도 하락.
- 수정 대상: `app/page.tsx`
- 제안:
  - 게스트: 히어로 + 핵심 CTA 2개(로그인, 마켓보기) 우선 배치
  - 로그인 폼은 접기(아코디언) 또는 `/login` 유도 중심으로 단순화

### P1 (이번 스프린트)
3. 상단 내비게이션 모바일 대응 부족
- 현상: 모바일에서 상단 버튼이 모두 한 줄/여러 줄로 노출되어 밀도 높음.
- 문제: 터치 타깃 간격/시선 집중도 저하.
- 수정 대상: `components/AppNav.tsx`
- 제안:
  - 모바일에서 `홈/단어장/마켓`만 1차 노출
  - `오프라인/요금제/계정`은 `더보기` 메뉴로 축약

4. 마켓 페이지 리스트 스캔 비용 과다
- 현상: 카드 수가 많고 정보량이 커서 첫 진입 스캔 난이도 높음.
- 문제: 검색/필터 전환 전 이탈 가능성.
- 수정 대상: `app/wordbooks/market/page.tsx` 관련 컴포넌트
- 제안:
  - 카드 상단 메타(단어 수/난이도/다운로드)를 1줄 요약 배지화
  - 평점 0/리뷰 0 상태는 별점 UI를 회색 단일 지표로 축약
  - 추천 섹션을 접기 가능하게 처리

5. 푸터 사업자 안내 문구의 페이지 점유율 과대
- 현상: 모든 페이지 하단에 긴 사업자 문구가 반복 노출.
- 문제: 핵심 액션 완료 전 시각적 피로 증가.
- 수정 대상: `app/layout.tsx`
- 제안:
  - 기본은 1~2줄 요약 + `자세히 보기` 토글
  - 개발환경 경고 문구는 dev에서만 접힌 상태 기본

### P2 (다음 스프린트)
6. 메인-마켓-로그인 전환 퍼널 최적화
- 수정 대상: `app/page.tsx`, `app/login/page.tsx`, `app/wordbooks/market/page.tsx`
- 제안:
  - CTA 문구 표준화: `마켓 둘러보기`, `로그인하고 시작하기`
  - `next` 파라미터가 있는 로그인 유입 시 상단 안내 배너 노출
  - 로그인 실패/성공 후 돌아가는 경로 UX 문구 강화

7. 성능/체감 개선 (대형 리스트)
- 수정 대상: 마켓 리스트 렌더링 컴포넌트
- 제안:
  - 초기 렌더 카드 수 축소 + 점진 로딩
  - 필터/정렬 상태를 URL에 유지하되 클라이언트 계산 최소화
  - 스켈레톤 로딩 도입
