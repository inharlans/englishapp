# OpenCode Browser MCP 화면 점검 포인트

MCP 목록이 비어 있을 때, OpenCode Browser 화면 기준으로 빠르게 원인을 좁히는 절차입니다.

## 1) 세션 헤더 확인 (우측 상단 상태)
- `MCP` 영역에 서버명이 하나도 없는지 확인합니다.
- 같은 화면에서 연결 상태 배지(Disconnected/Degraded 등)가 보이면 우선 기록합니다.
- 서버명이 잠깐 나타났다가 사라지면 서버 프로세스 크래시 가능성이 큽니다.

## 2) 작업 폴더/레포 컨텍스트 확인
- 현재 세션의 작업 디렉토리가 `C:\dev\englishapp`인지 확인합니다.
- 다른 폴더에서 시작했으면 `.codex/config.toml`를 못 읽을 수 있습니다.
- 세션 시작 직후 로드된 프로젝트 정보에 `englishapp`가 표시되는지 확인합니다.

## 3) MCP 설정 파일 확인
- `.codex/config.toml`이 존재하는지 확인합니다.
- 서버 블록 문법 오타(따옴표/대괄호/배열 닫힘) 여부를 확인합니다.
- 현재 기준 최소 설정 예시는 다음과 같습니다.

```toml
[mcp.servers."chrome-devtools"]
command = "npx"
args = ["-y", "chrome-devtools-mcp@latest"]
```

## 4) 실행 환경 확인
- `npx` 실행 가능 여부를 먼저 확인합니다.
- 회사 보안 정책/프록시로 외부 패키지 다운로드가 차단되면 MCP 실행이 실패할 수 있습니다.
- 권한 승인 팝업이 있다면 허용했는지 확인합니다.

## 5) 세션 재시작 + 로그 확인
- 위 항목 수정 후 세션을 재시작합니다.
- 여전히 비어 있으면 MCP 시작 로그에서 실제 에러 메시지(권한/네트워크/명령어)를 확인합니다.
- 에러 메시지 1줄 요약 + 재현 조건을 문서에 남겨 후속 대응을 빠르게 합니다.

## 6) 정상 판정 기준
- 우측 상단 `MCP`에 최소 1개 서버명이 표시되면 연결 성공입니다.
- 연결 후 간단한 도구 호출(예: DevTools 열기/브라우저 페이지 확인)이 성공하면 운영 가능 상태입니다.
