# 퀴즈 피드백 및 러닝 루프 강화안

## 배경
- 현재 퀴즈는 `정답/오답`과 다음 버튼 중심으로 흐름이 빠르지만, 왜 맞았는지/틀렸는지 학습 피드백이 약함.
- 오답 후 복습 동작이 `오답 목록` 화면으로 분리되어 즉시 학습 루프가 끊김.

## 문제 정의
- 즉시 피드백의 학습 가치가 낮아 반복 학습 효율이 떨어짐.
- `정답 표시 -> 다음`만으로는 재발 방지 학습이 어려움.

## 목표
- 1문항 단위의 이해-교정-재확인 루프를 강화.
- 오답을 “실수”에서 “복습 자산”으로 전환.

## 제안
1. 오답 시 2단계 피드백
- 1단계: 정답/유사 정답/품사/핵심 용례 짧게 표시
- 2단계: `같은 문제 다시 풀기` 또는 `다음으로` 선택

2. 정답 시 최소 강화 피드백
- 정답이어도 핵심 의미 1줄 + 대표 예문 1개 표시(0.8초 자동 접힘 가능)

3. 세션 내 즉시 재출제 큐
- 오답 문항을 현재 파트 끝나기 전 1회 재노출.
- 재정답 시 `회복` 상태로 자동 전환.

4. 문항별 히스토리 칩
- 최근 N회 정답/오답 패턴을 간단히 표시(`YNNY`).
- 사용자가 체감 진척을 즉시 확인 가능.

## 구현 범위
- 퀴즈 결과 카드 UI
- 세션 큐 로직(오답 재출제)
- 상태 전이(`WRONG -> HALF -> CORRECT`) 정책 정리

## 수용 기준
- 오답 재출제 기능 동작.
- 평균 세션 정답률/회복률 개선.
- 사용자 체감 피드백 만족도 상승.
