# 퀴즈 품질 관측 및 자동 검증 체계안

## 배경
- 채점 신뢰성과 데이터 품질 이슈는 기능 수정만으로 재발 방지가 어려움.
- 운영 중 회귀를 조기에 감지할 계측/자동 검증 체계가 필요함.

## 문제 정의
- 어떤 유형의 오답이 “실력 부족”인지 “시스템 판정 문제”인지 구분이 어려움.
- 배포 후 품질 저하를 자동 탐지하기 어렵다.

## 목표
- 퀴즈 품질을 지표로 관리하고, 회귀를 CI/운영 단계에서 차단.

## 제안
1. 채점 진단 이벤트 추가
- 문항 ID, 사용자 입력(마스킹), 정규화 결과, 매칭 단계(1차/2차/3차), 최종 판정 기록.

2. 품질 지표 대시보드
- `이의 가능 오답률`(정답 후보와 유사했지만 오답 처리된 비율)
- 단어장별 비정상 오답 집중 구간
- 품사/의미 비문 탐지율

3. 리그레션 테스트 세트 구축
- 대표 논쟁 케이스(예: 전치사 의미 다중 표현) 고정 샘플.
- CI에서 채점 결과 snapshot 비교.

4. 운영 플래그/롤백
- 새 채점 로직을 feature flag로 단계 배포.
- 이상 징후 시 즉시 이전 룰로 롤백 가능하게 구성.

## 구현 범위
- 퀴즈 채점 로깅
- 지표 수집 파이프라인
- CI 테스트 데이터셋/검증 스크립트

## 수용 기준
- 배포 후 채점 회귀를 CI에서 사전 탐지.
- 운영에서 이의 가능 오답률 추세 하락.
- 이슈 재현/원인 추적 시간이 단축.
