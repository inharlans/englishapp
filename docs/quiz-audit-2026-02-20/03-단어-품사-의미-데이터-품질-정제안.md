# 단어 품사/의미 데이터 품질 정제안

## 배경
- 오답 목록 실점검에서 `부사을수록`, `대명사짐`, `동사을게` 등 비자연 표기가 관측됨.
- UI 이슈가 아니라 원천 데이터 정규화/품질 관리 이슈로 판단됨.

## 문제 정의
- 품사와 의미 결합 규칙이 일관되지 않아 학습 신뢰도가 저하됨.
- 데이터 품질 편차가 채점 신뢰성 문제와 연결됨.

## 목표
- 사용자 노출 품질을 “자연스러운 한국어 학습 문구” 기준으로 상향.
- 데이터 생성/수정 이후 품질 회귀를 방지.

## 제안
1. 품질 규칙(Validator) 명문화
- 금지 패턴: `품사+조사 비문`, 분절 오류(`을게`, `짐`), 의미 없는 기호 조합.
- 필수 규칙: 품사 태그 유효성, 의미 최소 길이, 중복/유사 의미 병합.

2. 자동 정제 파이프라인
- 1차 정규식 교정(명백한 노이즈 제거)
- 2차 사전 기반 치환(표준 품사/의미)
- 3차 LLM 보조 교정(검토 로그 필수)

3. 검수 등급 도입
- A: 자동 통과
- B: 자동 교정 후 통과
- C: 수동 검수 필요

4. 운영 대시보드
- 단어장별 품질 점수, 비문 비율, 최근 정제 이력 표시.

## 구현 범위
- 데이터 정제 스크립트/배치
- 사전 파일(품사/의미 표준 매핑)
- 관리자 검수 워크플로우

## 수용 기준
- 주요 공개 단어장 비문율 목표치 이하(예: 1% 미만).
- 퀴즈 노출 문구 자연성 개선 확인.
- 신규 업로드 단어장 자동 품질 게이트 적용.
