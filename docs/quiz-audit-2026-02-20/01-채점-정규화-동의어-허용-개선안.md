# 단어/의미 퀴즈 채점 정규화 및 동의어 허용 개선안

## 배경
- 실서비스 점검(2026-02-20)에서 의미 퀴즈 `at` 문제에 `에서`를 입력했는데 오답 처리됨.
- 현재 정답 문자열은 `at / (전)...로, ...에, ...에게, ...에서, ...에게로`처럼 복합 포맷으로 보여, 사용자 입력과 내부 정답 비교 기준이 불명확함.

## 문제 정의
- 허용 가능한 의미 변형(조사/띄어쓰기/부호/접두 표현) 처리 기준이 약함.
- `정답` 판정의 설명 가능성이 낮아 사용자가 오답 사유를 신뢰하기 어려움.

## 목표
- 정답 판정 일관성 확보.
- 실제 학습 의도(핵심 의미 이해) 중심으로 채점 유연성 강화.

## 제안
1. 입력 정규화 레이어 도입
- trim, 소문자화, 연속 공백 축소, 특수문자/생략부호 정리.
- 한국어 조사형 표준화(예: `...에서`, `에서`를 동일 후보군으로 정리).

2. 정답 사전 구조 개선
- 단일 문자열이 아닌 `canonical + aliases[]` 구조로 저장.
- 예시:
  - canonical: `에서`
  - aliases: `...에서`, `~에서`, `에서(장소)`

3. 허용 매칭 단계화
- 1차: exact normalized match
- 2차: alias exact match
- 3차: 안전한 포함/형태소 기반 약일치(오탐 방지 룰 포함)

4. 오답 사유 메시지 개선
- `입력: X / 허용 정답 예: A, B, C` 형태로 즉시 안내.
- 오답일 때 “채점 기준 보기” 링크 제공.

## 구현 범위
- 퀴즈 채점 API/로직
- 단어 데이터 스키마(정답 후보 다중화)
- 운영용 정답 사전 보정 스크립트

## 수용 기준
- 대표 케이스(`at`-`에서`) 정답 처리.
- 회귀 테스트에서 기존 정답률 하락 없음.
- 채점 관련 CS/오답 신고 건수 감소.

## 우선순위
- P0 (즉시 적용 권장)
