# 퀴즈 진행 동선 및 모바일 UX 개선안

## 배경
- 퀴즈/목록 화면에서 파트 버튼(1~20)이 길게 노출되어 탐색 밀도가 높음.
- 모바일 뷰(390x844)에서도 동일 패턴으로 정보량이 과다해 핵심 학습 영역 집중이 약해짐.

## 문제 정의
- 파트 네비게이션이 메인 퀴즈 인터랙션보다 시각 우선순위가 높음.
- 작은 화면에서 입력-제출-피드백 루프 집중도가 떨어짐.

## 목표
- “문제 풀이” 중심 레이아웃으로 재정렬.
- 모바일 우선 UX에서 조작 수를 줄이고 집중도를 높임.

## 제안
1. 파트 선택 UI 간소화
- 기본: `현재 파트`, `이전/다음`, `파트 선택 드로어`로 축소.
- 고급 사용자만 펼칠 수 있는 `전체 파트 보기` 제공.

2. 상단 고정 학습 바
- 진행률, 현재 파트, 남은 문항, 오답 수만 요약 표시.

3. 입력 영역 우선 배치
- 문제/입력/제출/결과를 한 덩어리 카드로 고정.
- 결과 확인 후 자동으로 다음 입력 포커스 이동.

4. 모바일 전용 밀도 프리셋
- 기본값: `집중` 또는 `표준`으로 통일.
- 비핵심 패널(설정/탭)을 접힘 상태로 시작.

## 구현 범위
- 퀴즈 공통 레이아웃 컴포넌트
- 파트 네비게이션 컴포넌트 재설계
- 반응형 브레이크포인트별 기본 상태 정의

## 수용 기준
- 모바일에서 첫 입력까지 탭/스크롤 수 감소.
- 세션 이탈률 감소, 평균 풀이 문항 수 증가.
- 사용자 피드백에서 “복잡함” 관련 불만 감소.
