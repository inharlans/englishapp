# MCP 1분 점검표 (4단계)

OpenCode Browser 우측 상단 MCP 목록이 비어 있을 때, 아래 4가지만 빠르게 확인합니다.

## 1) 세션 재시작
- 현재 세션을 닫고 새 세션을 시작합니다.
- 새 세션에서 MCP 목록이 다시 보이는지 먼저 확인합니다.

## 2) 설정 경로 일치 확인
- 현재 작업 디렉토리와 MCP 설정 파일 경로가 같은 프로젝트를 가리키는지 확인합니다.
- `englishapp` 기준 설정 파일은 `.codex/config.toml`입니다.

## 3) MCP 서버 실행 가능 여부 확인
- MCP 서버 실행 명령(`npx ...`)이 이 환경에서 실행 가능한지 확인합니다.
- 권한 승인 팝업이 있었다면 거절되지 않았는지 확인합니다.

## 4) 실행 로그 확인
- MCP 목록이 비어 있으면 서버 시작 실패 로그를 확인합니다.
- 실패 원인(권한, 네트워크, 명령어 오류)을 정리한 뒤 세션을 다시 시작합니다.

## 판정 기준
- 우측 상단 MCP 목록에 서버명이 표시되면 정상 연결입니다.
- 목록이 계속 비어 있으면 해당 세션에서는 MCP 도구를 사용할 수 없는 상태입니다.
