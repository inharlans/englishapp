# 오프라인 데이터 재동기화 계획

## 목표

- 과거에 저장된 오염 오프라인 데이터를 정리해 fallback 의존도를 낮춘다.

## 실행 항목

1. 오프라인 저장 시 버전 해시 포함
2. 앱 진입 시 버전 불일치면 재다운로드 유도
3. `의미 데이터 점검 중입니다` 표시 항목에 “동기화” CTA 제공
4. 백그라운드 마이그레이션(IndexedDB 레코드 정제)

## 완료 기준

- 오프라인 모드에서 fallback 노출률 1% 미만
