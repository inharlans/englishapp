# 관측성 SLO/알림 계획

## 목표

- 장애 감지 시간을 줄이고 복구 속도를 높인다.

## SLO 초안

1. 핵심 API 성공률 99.5% 이상
2. 크론 성공률 99% 이상(7일 이동평균)
3. P95 응답시간 500ms 이하(핵심 경로)

## 실행 항목

1. API별 에러코드 표준화(`CONFIG/AUTH/DB/UPSTREAM`)
2. 크론 실패 시 즉시 알림(Slack 또는 이메일)
3. `/admin`에 SLO 위반 배지/최근 위반 내역 표시
4. 주간 리포트 자동 생성

## 완료 기준

- 실패 원인 파악에 필요한 시간 10분 이내
