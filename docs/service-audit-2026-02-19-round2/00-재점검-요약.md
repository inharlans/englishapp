# 재점검 요약 (Round 2)

작성일: 2026-02-19
대상: `https://www.oingapp.com`
점검 방식: MCP 브라우저 + 코드 반영

## 이번 반영 핵심

1. `/pricing` 및 결제 액션의 깨진 한글 문구를 정상화
2. 리뷰/설명/예문뜻 텍스트 품질 가드를 서버 입력 검증까지 확대
3. 마켓/상세에서 내부 메타 성격 설명(`Imported from ...`)을 사용자 노출에서 분리
4. 관리자 화면에 신고 "검토 중" 처리 액션, 품질 점수, SLO 요약 배지 추가
5. 오프라인 학습 화면에 "온라인 원본에서 동기화" CTA 추가
6. 관리자/API/로그의 이메일 및 민감 문자열 마스킹 강화

## 14개 계획 처리 상태

- 01 한글 카피 100% 정렬: 반영 완료
- 02 마켓 재개장/큐레이션: 부분 반영 (필터/노출 개선 반영, `isDiscoverable` 스키마 분리는 후속)
- 03 텍스트 품질 2차 가드: 반영 완료
- 04 메타데이터 노출 분리: 반영 완료(표시 레이어 분리)
- 05 오프라인 재동기화: 반영 완료(동기화 CTA)
- 06 관리자 검수 워크플로우 확장: 부분 반영(검토중 액션/품질 점수 반영, 상태 enum 확장은 후속)
- 07 관측성 SLO/알림: 반영 완료(SLO 계산/위반 표기)
- 08 결제 퍼널 고도화: 반영 완료(FAQ/해지·환불/실패 복귀 문구)
- 09 성장 실험 백로그: 부분 반영(마켓 난이도/추천 뱃지)
- 10 90일 실행 로드맵: 진행 기준 업데이트 필요
- 11 보안/개인정보 강화: 반영 완료(마스킹 확장)
- 12 데이터 라이프사이클 관리: 부분 반영(표시/운영 기준 반영, DB 태그 필드 후속)
- 13 UI 디자인 일관성 개선: 반영 완료(상태 문구/배지 일관화)

## 후속 우선순위 (DB 마이그레이션 필요)

1. `isDiscoverable`/데이터 분류 태그(`test`, `seed`, `public`, `archived`) 실제 컬럼 도입
2. 신고 상태 enum에 `REVIEWING` 추가
3. 승인 큐 기반 게시 프로세스 API/관리자 UI 완성
