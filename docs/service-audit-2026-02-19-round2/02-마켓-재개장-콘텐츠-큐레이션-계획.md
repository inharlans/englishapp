# 마켓 재개장/콘텐츠 큐레이션 계획

## 목표

- “테스트 데이터 노출 차단”과 “마켓 비어있음 방지”를 동시에 달성한다.

## 문제

- 현재 필터가 강해 `총 0개`가 될 수 있음
- seed 계정 콘텐츠가 서비스 노출 정책과 충돌

## 실행 항목

1. `isDiscoverable` 필드 도입(노출 여부를 명시적으로 관리)
2. 운영 승인된 단어장만 discoverable=true로 승격
3. 초기 런칭용 큐레이션 세트(10~20개) 별도 제작
4. 필터 조건을 “키워드 기반”에서 “승인 기반”으로 전환

## 완료 기준

- 마켓 기본 페이지 상시 노출 카드 12개 이상
- 테스트/내부 데이터 노출 0건
