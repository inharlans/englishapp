# 1. 미세 인터랙션을 줄이고 의미 있는 전환만 남긴 ‘저소음 모션’ 디자인을 적용한다.

## 2. 문제점 설명
모든 요소가 움직이면 오히려 피곤하고 산만해집니다. 학습 앱에서 과한 모션은 정보 전달보다 시선 분산을 유발할 수 있습니다. 반대로 전환 피드백이 너무 없으면 조작 확신이 떨어집니다.

## 3. 제안된 해결 방안
- 버튼 hover/press, 탭 전환, 카드 등장에만 제한적으로 모션 적용
- 정답/오답 피드백은 색상 변화 + 짧은 스케일 애니메이션으로 통일
- 목록/페이지 이동은 skeleton + fade-in으로 로딩 체감 완화
- `prefers-reduced-motion` 환경에서는 애니메이션 강도 자동 축소

## 4. 예상 효과
UI가 불필요하게 요란하지 않고 안정적으로 느껴져 장시간 사용에 유리해진다. 필요한 순간에만 피드백을 주기 때문에 조작 확신은 높이고 피로는 낮출 수 있다. 결과적으로 미적 완성도와 기능적 명확성이 동시에 좋아져 학습 집중력을 유지하기 쉬워진다.
