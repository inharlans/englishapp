# 1. 빈 상태(Empty State)를 행동 중심으로 재설계해 막힘 없는 탐색 흐름을 만든다.

## 2. 문제점 설명
데이터가 없을 때 단순 텍스트만 보이면 사용자는 다음 행동을 떠올리기 어렵습니다. 특히 첫 사용자는 화면이 비어 있으면 기능이 고장 났다고 오해할 수 있습니다. 빈 상태는 곧 제품 안내의 핵심 접점입니다.

## 3. 제안된 해결 방안
- 각 빈 상태에 한 줄 설명 + 1차 행동 버튼 + 보조 링크를 기본 템플릿화
- 예: 다운로드 없음 → `마켓 둘러보기`, 퀴즈 항목 없음 → `암기 먼저 시작`
- 간단한 일러스트/아이콘을 추가해 감정 톤을 부드럽게 구성
- 최근 행동 기반 추천 링크(최근 본 단어장 이어서)를 선택 노출

## 4. 예상 효과
빈 화면이 안내 화면으로 바뀌어 사용자가 다음 행동을 즉시 이해하고 이동할 수 있다. 첫 사용자 혼란이 줄고, 중간 이탈 지점에서 재유입 동선을 제공해 전환율이 개선된다. 기능이 없다는 인상 대신 친절한 흐름이 형성되어 제품 이미지와 사용 만족도가 함께 높아진다.
