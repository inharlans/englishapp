# 1. 브랜드 톤을 살리는 컬러 시스템과 타이포 스케일을 도입해 화면 일관성을 만든다.

## 2. 문제점 설명
페이지별로 색상과 텍스트 크기 인상이 조금씩 다르면 전체 앱이 조립식처럼 보일 수 있습니다. 작은 시각 차이 누적은 완성도를 낮추고 피로도를 높입니다. 특히 학습 앱에서는 반복 노출되는 UI의 일관성이 신뢰감에 직접 영향을 줍니다.

## 3. 제안된 해결 방안
- `primary/neutral/success/warn` 중심의 4축 컬러 토큰을 CSS 변수로 고정
- 제목/본문/보조텍스트를 6단계 타이포 스케일로 통일
- 버튼, 배지, 카드의 radius/그림자 값을 디자인 토큰으로 일원화
- 핵심 화면(라이브러리/상세/학습)의 시각 규칙을 컴포넌트 가이드로 문서화

## 4. 예상 효과
브랜드 톤이 통일되면 앱이 더 정돈되고 신뢰감 있게 느껴진다. 사용자는 화면이 바뀌어도 시각 패턴을 다시 학습하지 않아 인지 비용이 줄고, 반복 학습 상황에서도 피로가 낮아진다. 개발 측면에서도 토큰 기반 변경이 가능해 확장 시 품질 편차를 줄일 수 있다.
